3 ways we could build this

Each approach gets you a working app — the difference is how much it connects to your existing tools

A

⭐ Start standalone, add Teamleader later Recommended

Phase 1: Build the full app — Kanban, survey form, PDF, email templates — but Florian enters the project details manually. Works immediately, no API setup needed.

Phase 2: Connect Teamleader so deals flow in automatically. The app is already built, we just add the sync.

Pros

  • Working app in the shortest time
  • Can show it to others immediately
  • Teamleader integration as an upgrade, not a blocker

Cons

  • Florian enters project info twice until Phase 2 is done
B

🔗 Full Teamleader integration from day one

We connect to Teamleader's API first. The Kanban automatically fills with deals from the right pipeline stages. Florian never enters project info manually — it's always synced.

Pros

  • No double data entry at all
  • Always in sync with what's in Teamleader

Cons

  • Longer before anything works (OAuth setup, API testing)
  • If Teamleader API has issues, app breaks
C

🎯 Survey form only — solve the core pain first

Skip the Kanban entirely for now. Just build the survey form, photo upload, PDF generation, and email templates. The simplest possible tool that solves Florian's biggest headache today.

Pros

  • Fastest to build and ship
  • Immediately useful

Cons

  • No project overview — Florian still uses Teamleader for that
  • Not as impressive to demo
