Tool-by-tool breakdown

What we build natively, what we link, and what belongs in a later phase

✅ Replace natively — already planned or straightforward to build
iLovePDF ❌ Gone. We generate all PDFs natively from the survey form. No need to leave the app.
Outlook (email) ❌ Gone. Replaced by the 3 built-in email templates (appointment / report / changes).
ESDECK calculator ✅ Build natively. It's a form with formulas — we replicate it as a built-in calculator tab.
Flat roof distance calculator ✅ Build natively. Same idea — a calculator with inputs and an output. Straightforward.
🔗 Integrate as a link or embed — too complex to replicate, but keep it inside the app
Solaredge Designer 🔗 External link. A solar panel design tool this complex would take a year to rebuild. We add a button that opens it in a new tab, linked directly from the project.
WebODM (3D drone simulations) 🔗 External link. Same — too complex to replicate. Button in the project opens it directly.
GEOPUNT 🔗 Embed or deep link. We can open GEOPUNT pre-loaded with the project address so Florian lands on the right location automatically.
Google Calendar 🔗 Embed. Show Florian's Google Calendar in a panel inside the app so he can check availability without switching tabs.
⏳ Phase 2 — important but builds on the foundation
Teamleader Already planned. API sync so deals flow into the Kanban automatically.
Sharepoint (document storage) Link to specific project folders. Florian can open the right Sharepoint folder from within a project card. Full two-way sync is complex — link first.
Product database (with dimensions) Admin panel where products are added with specs. Referenced in the survey form and calculators. Solid Phase 2 feature.
🚀 Phase 3 — ambitious, powerful, but needs the rest built first
AI agent linked to GHS Sharepoint database This is genuinely exciting — an AI assistant Florian can ask "what are the dimensions of X panel?" or "how do I install Y on a flat roof?" and it searches the Sharepoint docs to answer. Fully buildable, but needs the product database and Sharepoint integration in place first.

Does this split make sense? Anything you'd move between phases?
